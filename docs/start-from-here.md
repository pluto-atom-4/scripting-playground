# Learning Guide Practical Shell Scripting

This learning plan focuses on automating, monitoring, and managing large-scale web infrastructure, tailored for a University of Washington IT (UW-IT) Software Engineer interview. Key daily exercises include utilizing grep, sed, awk, and curl to simulate managing 29,000+ websites, with an emphasis on log parsing, WordPress maintenance, and SQL backup automation.

## The job description 


### Key points — role & impact

- Position: Software Engineer supporting UW Shared Web Hosting and related infrastructure; high-impact role serving ~29K websites across UW and partners.
- Primary focus: maintain/customize software, integrate third-party systems (APIs/ETL), and operate/optimize shared web hosting environment.
- Scale & visibility: production services used by faculty, staff, students, patients; outages affect critical university operations.

### Responsibilities (condensed)

- UW Shared Web Hosting support/maintenance (~40%)
- Unix/Linux engineering and full-stack operational support (Kubernetes, managed services) (~20%)
- Automation, deployment tooling, config management, patching (10–20%)
- Team participation, on-call rotation, stakeholder communications, occasional leadership/hiring input

### Required skills & experience

- BS in CS or equivalent + ~4 years relevant experience (systems administration, support, deployments)
- Experience with large web hosting environments (10,000+ sites)
- Databases: SQL Server, PostgreSQL and/or MySQL
- Shell scripting (Unix), familiarity with APIs and system integrations
- Ability to lead/influence without direct authority; documentation and project management skills
- Customer-service orientation; strong verbal/written communication

### Helpful / desired skills

- LAMP stack: Linux, Apache, MySQL, PHP; WordPress administration (themes/plugins)
- Perl, JavaScript, PowerShell
- Ansible, Kubernetes, container experience
- ITSM/ServiceNow, SSRS, accessibility compliance
- Academic/research environment experience

### Technical themes & expectations

- Emphasis on operational reliability, automation, and reducing technical debt.
- Mix of engineering (development/configuration), systems administration, and service management.
- Cross-functional collaboration with non-technical stakeholders; regular reporting.

### Risks & challenges
- High operational burden: large scale shared hosting + on-call requirements.
- Must balance maintenance, incident response, new feature/project work.
- Requires broad full-stack ops skillset; potential for fragmented work across many platforms and languages.

### Fit profile — who would succeed

- SRE/DevOps or full-stack systems engineer with LAMP and Linux background, strong scripting and DB experience, comfortable supporting many sites and users, and who prefers operationally-focused engineering plus stakeholder interaction.

### Interview prep checklist (recommended)

- Examples of troubleshooting large hosting outages and incident postmortems
- Scripts/automation you've built for deployment/patching/config management (Ansible, custom tooling)
- Experience with MySQL/Postgres and scaling/backup/restore scenarios
- WordPress hosting experience and PHP/Perl/JS examples if available
- Stories showing cross-team influence, documentation, and customer service
- Availability for on-call and hybrid (Seattle) schedule
